# DispatchService

This service contains endpoints for dispatch functions on asset/truckload movements.

## Operations

name | role | description  
---|---|---  
[GET /dispatch/checkTrailerPreassignments/{trailerId}](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=checkTrailerPreassignments&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Checks to see if the trailer has existing preassignments other than the current movement being checked for dispatch.  
[GET /dispatch/dispatchControl](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=getDispatchControl&role=-1) |  [](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Returns the user's current company's dispatch control record.  
[GET /dispatch/emptiesToCorrect](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=getEmptiesToCorrect&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Determines previous empties for a given tractor, when canceling a movement.  
[POST /dispatch/assign/{movementId}](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=assignEquipment&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Assigns/updates assignments of equipment for the given movement.  
[POST /dispatch/cancel/{movementId}](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=cancelMovement&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Cancels an in-progress or delivered movement, given it is the current movement for the driver.  
[POST /dispatch/changeEquipment/{movementId}](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=changeEquipment&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles equipment ID changes for a given movement and specific piece of equipment.  
[POST /dispatch/clearPreassignments/{movementId}](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=clearPreassignments&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Clears all preassigned equipment for the given movement.  
[POST /dispatch/clearStop/{stopId}](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=clearStop&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Clears the stop with the given dates and creates a service failure record if necessary.  
[POST /dispatch/locateTractor](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=locateTractor&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Locates a tractor at the provided location.  
[POST /dispatch/moveTrailer](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=moveTrailer&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Moves the trailer to the provided location.  
[POST /dispatch/reconsign/{movementId}](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=reconsign&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Reconsigns the order for the given movement.  
[POST /dispatch/split/{splitBeforeStopId}](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=splitMovement&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Splits the given movement, given the specified stop and equipment information.  
[POST /dispatch/unsplit/{movementId}](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=unsplitMovement&role=-1) |  [Users](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Unsplits the given movement.  
[POST /dispatch/{stopId}/acceptOffer](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleDriverOfferAcceptance&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver accepts an offered stop. If the {@link DriverStopReport#getAppliesToGroup()} variable is `true`, then this stop plus others in the same group will be accepted together. See {@link RowStop#buildGroupingKey()} for what constitutes a group of stops. Currently, this only works for LTL movements.  
[POST /dispatch/{stopId}/arrive](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleDriverArrival&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver arrives at a stop. If the {@link DriverStopReport#getAppliesToGroup()} variable is `true`, then this stop plus others in the same group will be marked arrived together. See {@link RowStop#buildGroupingKey()} for what constitutes a group of stops. Currently, this only works for LTL movements. In the future, we will use this method for drivers to report their arrivals on truckload movements as well.  
[POST /dispatch/{stopId}/beginTrip](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleBeginTrip&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver begins a trip. Currently, this only works for LTL peddle movements since these are the only ones that use begin and end trip stops.  
[POST /dispatch/{stopId}/complete](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleDriverCompletion&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver completes a delivery or pickup stop. Currently, this only works for LTL movements. In the future, we will use this method for drivers to report their starts on truckload movements as well.  
[POST /dispatch/{stopId}/createServiceFailure](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleDriverServiceFailure&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver reports a service failure. If the {@link DriverStopReport#getAppliesToGroup()} variable is `true`, then this stop plus others in the same group will all receive service failure records. See {@link RowStop#buildGroupingKey()} for what constitutes a group of stops. Currently, this only works for LTL movements. In the future, we will use this method for drivers to report their arrivals on truckload movements as well.  
[POST /dispatch/{stopId}/declineOffer](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleDriverOfferDecline&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver declines an offered stop. If the {@link DriverStopReport#getAppliesToGroup()} variable is `true`, then this stop plus others in the same group will be declined together. See {@link RowStop#buildGroupingKey()} for what constitutes a group of stops. Currently, this only works for LTL movements.  
[POST /dispatch/{stopId}/depart](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleDriverDeparture&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver departs from a stop. If the {@link DriverStopReport#getAppliesToGroup()} variable is `true`, then this stop plus others in the same group will be marked departed together. See {@link RowStop#buildGroupingKey()} for what constitutes a group of stops. Currently, this only works for LTL movements. In the future, we will use this method for drivers to report their arrivals on truckload movements as well.  
[POST /dispatch/{stopId}/endTrip](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleEndTrip&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver ends a trip. Currently, this only works for LTL peddle movements since these are the only ones that use begin and end trip stops.  
[POST /dispatch/{stopId}/freightPhoto](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleDriverFreightPhoto&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver submits a photo of freight. Currently, this only works for LTL movements.  
[POST /dispatch/{stopId}/scanDocument](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleDriverDocumentScan&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver scans documents. Currently, this only works for LTL movements.  
[POST /dispatch/{stopId}/start](https://tms-cfaa.loadtracking.com:5690/ws/docs/services?service=DispatchService&operation=handleDriverStart&role=-1) |  [Drivers](https://tms-cfaa.loadtracking.com:5690/ws/docs/roles?role=-1) | Handles the changes needed to be made when a driver starts delivering at a stop. Currently, this only works for LTL movements. In the future, we will use this method for drivers to report their starts on truckload movements as well.
